# 真机联调问题记录

> 设备: Xiaomi 24115RA8EC / Android 15 / MIUI / 1220x2712
> 日期: 2026-05-29 ~ 2026-05-30

---

## 一、已解决的问题

### 1. scrcpy 投屏连接
- **现象**: scrcpy 反复崩溃，watchdog 每 5 秒重启一次
- **原因**: V5.0 隐匿隧道 `adb forward localabstract` 与 scrcpy 3.1 不兼容，scrcpy 连不上 server
- **修复**: `stealth_mode` 默认改为 `False`，走 scrcpy 原生连接。隐匿隧道留作后续优化
- **文件**: `src/core/adb_controller.py`

### 2. 中文输入 — ADBKeyboard 广播被 MIUI 拦截
- **尝试过但失败的方案** (共 11 种):
  - `adb shell input text` — 仅 ASCII
  - `cmd clipboard set` — Xiaomi 未实现
  - `service call clipboard` — Parcel 不兼容
  - Taobao Deep Link — 未触发搜索
  - scrcpy 剪贴板同步 + Ctrl+V — 未同步
  - pyautogui 打字到 scrcpy 窗口 — 键盘事件未转发
  - scrcpy UHID 键盘模式 — 窗口激活报错
  - scrcpy `--text` 参数 — v3.1 已移除
  - scrcpy MOD+Shift+V — 未生效
  - `cmd input text` — Xiaomi 不支持
  - ADB 硬件键值逐键拼音 — 搜狗 IME 不转中文
- **最终方案**: ADBKeyboard 一次性 broadcast 整句 + 先清空搜索框
  ```bash
  adb shell ime set com.android.adbkeyboard/.AdbIME
  adb shell am broadcast -a ADB_INPUT_TEXT --es msg '朱公子'
  ```
- **文件**: 命令行直接调用

### 3. 进店路由 — 搜索结果页找到店铺入口
- **现象**: 搜索后不知点哪里进店，OCR 多次定位到商品卡片脚注而非店铺入口
- **发现**: 搜索结果页有多个同名文字（搜索框、商品卡片脚注、店铺入口卡片），Y 坐标越大越靠下、越可能是店铺入口
- **解决方案**: 同名文字取 Y 最大的（最底部）点击
- **文件**: `tap_store.py`

### 4. Android 15 长内容分享
- **发现**: Android 15 对 `am broadcast` 的 Intent extras 有 1MB 大小限制
- **影响**: 发送大量 base64 图片数据时可能截断
- **解决方案**: 直接使用 `requests.post` JSON body，绕过广播限制
- **文件**: `src/ocr/vlm_extractor.py`

### 5. 滑动方向
- **现象**: 进店后往下滑没反应
- **原因**: 店铺入口在页面最底部（y=2152），进店后页面也在底部，往下滑无内容
- **修复**: 先进店后先上滑 3 次回到顶部，再下滑浏览商品
- **文件**: `topology_scan.py`

### 6. 卡片切分 — 固定网格 24→10 款
- **现象**: 固定 2×4 网格切出 24 款，远超实际 11 款，pHash 重复
- **原因**: 固定网格切到广告横幅、空白区域、非商品 UI
- **修复**: 
  - 轮廓检测 (`CardSplitter` contour 模式)
  - Y 轴安全区 (18%-90%) 拒绝残缺卡片
  - 形态学膨胀连接边缘断点
- **文件**: `src/consumer/card_splitter.py`

### 7. Ollama + qwen2.5:7b 文本模型用于标题筛选
- **现象**: OCR 抓到的全是营销标签，不是真实商品名
- **方案**: OCR 所有候选文字 → 发给 qwen2.5:7b → 模型挑出商品名
- **问题**: 文本模型无视觉能力，无法看图判断；`num_predict=48` 太小导致标题截断
- **当前**: `num_predict=128`，标题质量部分改善，仍不稳定

## 二、仍在进行中的问题

### 1. 卡片切分精度不足
- **现象**: 轮廓检测只切出 3 张（大块主图），网格切出 10 张但用户确认只有前 2 张正确
- **根因**: 该店铺版面不是标准 2 列网格，卡片大小不一、排列不规则
- **待做**: 需要针对具体版面调整切分参数，或用 OCR 动态定位每个卡片的边界

### 2. 标题提取准确率低
- **现象**: OCR+LLM 提取的 10 条标题，用户反馈"没有一个对上的"
- **根因**: 
  - 卡片切不准 → 标题区定位错
  - 该店版面不同于标准淘宝商品卡布局
  - OCR 优先抓取大字号营销词，真实标题字号小、颜色浅
- **待做**: 物理分区方案（用户建议的 0.65-0.80 标题区），确认卡片切对后再试

### 3. 淘宝反爬 — 自动化搜索返回空结果
- **现象**: ADBKeyboard 一次性注入 7 个字，搜索返回"搜索结果竟然是空的"
- **推测根因**: 
  - 盲区一：没点"搜索店铺"联想下拉，直接回车导致搜了"宝贝"域
  - 盲区二：文字瞬间注入（非人类击键速率），触发静默熔断
- **当前规避**: 人工手动搜，后续自动化接管进店后的流程

### 4. 仿生打字机实现路径
- **计划**: 废弃一次性 broadcast，改为逐字发送 + 生物级停顿 + 点击联想词
- **障碍**: ADBKeyboard 逐字 broadcast 会互相覆盖（每次替换而非追加）
- **待验证**: ADB 硬件键值逐键 + 搜狗拼音选词

### 5. 触控 DOWN/UP 防误触延时
- **文档要求**: DOWN 与 UP 之间间隔 50-120ms
- **当前状态**: 使用 `adb shell input tap` 一键完成，无间隔控制
- **影响**: 可能被 Android 防误触机制拦截，导致"点了没反应"
