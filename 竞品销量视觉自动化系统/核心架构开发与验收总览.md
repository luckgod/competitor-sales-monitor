# 核心架构开发与验收总览 (V5.0 生产级里程碑版)

> 项目整体落地划分为五个纵向阶段，每个阶段必须通过"地狱级拦截测试（Chaos Test）"方可进入下一阶段。
> V5.0 核心升级：微观实时订单流水捕获 + Qwen2.5-VL 多模态提取 + 24小时智能熔断截断 + 动态入口寻址。

## 阶段一：底层通信与链路自愈保障层（环境与骨架）

> 打通物理设备与控制端的无状态连接，建立环境断言闸门与内核隐匿防线。

### 1.1 开发执行步骤

**环境强断言预检闸门（V5.0）：**
- 连接唯一性断言：`adb devices` 仅识别一台合法设备。
- 屏幕状态自愈：锁屏/熄屏自动唤醒并滑动解锁。
- App 版本匹配断言：VersionCode 与 YAML 坐标模板一致性校验。
- 底层隐匿校验：检测 KernelSU / APatch 内核防护状态。

**物理环境常驻配置：**
- 配置 ADB 环境变量，调教 Android 测试机（USB调试 + 安全设置 + 充电不熄屏）。
- 部署 KernelSU 或 APatch，确保反爬 SDK 无法扫描调试特征。

**无特征投屏管道流挂载（V5.0 升级）：**
- 源码级去特征化 scrcpy：批量重命名 `scrcpy`/`scrcpy-server` 为系统级包名。
- Local Socket 使用 UUID 动态随机生成，用完即毁。
- 1080P / 15fps 限制，H.264 帧以内存管道推送到 Python 进程。

**断线自愈守护进程（Watchdog）：**
- 独立常驻健康监控线程，5 秒间隔轮询 scrcpy PID 与 ADB 连接状态。
- 断连检测 → `adb kill-server && adb start-server` → `wait-for-device` → 重拉 scrcpy。
- 连续 3 次失败 → 发送远程告警。

**运行期断点状态持久化（Session Checkpoint）：**
- 实时记录当前 `store_id`、商品 Index 锚点、`last_successful_virtual_id`。
- 支持 Ctrl+C 优雅退出 → 残留帧排空 → 序列化 `checkpoint.json`。
- 下次插线启动 → 交互提示续跑 → 自动跳转至断点坐标。

### 1.2 单元验收标准

**测试算子 1（预检断言测试）：**
- 模拟插入未解锁手机 → 验证系统红字报错退出。
- 模拟 KernelSU 未激活 → 验证系统红字报错退出。
- 模拟 App 版本与配置模板不一致 → 验证系统熔断退出。

**测试算子 2（热插拔测试）：**
- 人为拔掉 USB 数据线，5 秒后重插。

**判定红线：**
- 主程序**绝对不允许抛出异常退出**。
- 看门狗在 **3 秒**内检测到断连并执行自愈握手。
- 重连后自动重拉 scrcpy，根据 Index 锚点续爬。

**测试算子 3（scrcpy 端口隐匿测试 — V5.0 新增）：**
- 手机端执行 `netstat -an` 和 `cat /proc/net/unix`。
- 验证无 `scrcpy` 特征字符串或固定端口暴露。
- 进程列表中投屏服务伪装为普通系统服务名。

## 阶段二：并发容器与背压控制框架层（骨骼与流水线）

> 解决手机高频截图与本地多模态大模型慢速推理之间的"算力不对称"矛盾。

### 2.1 开发执行步骤

**多线程并发模型：**
- 线程 A（生产者）：接收投屏视频流、视觉干扰剔除（直播悬浮窗动态涂白）、手势注入、向队列投递。
- 线程 B（消费者）：OpenCV 卡片切分、白块过滤、Qwen2.5-VL 多模态提取、时间归一化、24小时熔断判定。

**有界阻塞队列配置：**
- `max_size = 10`，禁止无界队列。
- 队列满 → 生产者 `WAITING`；队列 < 7 → `Notify` 唤醒。
- 防冻死：生产者阻塞超 30 秒 → 自动降级至 720P 或跳帧。

**硬件级触控属性仿真（V5.0 新增）：**
- 放弃 `adb shell input swipe`，直接向 `/dev/input/eventX` 写入二进制原始事件。
- 动态压力矩阵：DOWN(0.3) → MOVE(0.5~0.8 动态波动) → UP(0.2)。
- 触控面积随贝塞尔曲线正态形变。
- 2D 高斯热区点击：95% 点击集中在控件中心偏左下，边缘概率指数衰减。

**白天动态干扰视觉过滤：**
- 连续三帧像素变动率 > 15% → 判定直播/促销干扰 → 局部纯白遮罩涂白。

**行为熵干扰模型（V5.0 新增）：**
- 视觉回看（概率 8%）：下滑 3 屏后反向微幅回滚，停顿 2 秒。
- 兴趣误触（概率 5%）：随机进入详情页，3-7 秒停留后返回。

### 2.2 单元验收标准

**测试算子（极限背压与内存压力测试）：**
在线程 B 中注入 `sleep(5)`，生产者高速投递。

**判定红线：**
- 队列满 10 帧后手机滑动和截图**瞬间强制悬停**。
- 内存占用线**横向拉平**，严禁 OOM。
- 消费者恢复后自动恢复滑动。

**测试算子（触控 MotionEvent 审计 — V5.0 新增）：**
- 抓取连续 500 次滑动的 Pressure/Size 属性日志。
- 验证压力值在 0.3~0.8 区间动态波动（非固定值）。
- 触控面积随正态分布微调（非恒定为 0）。

**测试算子（点击热区空间分布 — V5.0 新增）：**
- 统计 1000 次对同一按钮的点击坐标分布。
- 验证呈高斯圆斑分布（非均匀矩形）。

## 阶段三：视觉解析、多模态提取与智能熔断（核心大脑）

> 完成 OpenCV 卡片切分、懒加载过滤，接入 Qwen2.5-VL 多模态大模型，实现动态入口寻址与 24 小时边界熔断。

### 3.1 开发执行步骤

**网格化 OpenCV 卡片剪切：**
- 对单帧大图执行轮廓检测，按相对坐标比例切分"主图区(A)"、"标题区(B)"、"销量区(C)"。

**懒加载（Lazy Loading）无效帧过滤：**
- 计算信息熵与灰度方差。熵 < 1.0 且方差 < 10 → 判定白块 → 微秒级拦截销毁，不触发大模型。

**动态入口寻址模块（V5.0 核心新增）：**
- 利用 PaddleOCR 在屏幕中下部检索流量探针标签：

```python
ENTRY_FEATURES = [
    r'超\d+人加购', r'\d+人感兴趣',
    r'近\d+天\d+\+人逛过', r'近期销量飙升',
]
```

- 命中后提取文本矩形中心坐标，叠加高斯偏移后点击唤醒"实时销量"弹窗。
- 未命中则降级抓取顶部常规"月销"数据后退出。

**视觉多模态数据提取（V5.0 核心变更）：**
- 放弃 OCR + Regex 两道式管道，全面升级为 **Qwen2.5-VL-7B-Instruct** 端到端语义提取。
- System Prompt 约束输出格式：

```json
{
  "summary": {"cart_adds": "...", "growth": "..."},
  "orders": [
    {"buyer": "...", "is_repeat": true/false, "sku": "...", "time_str": "..."}
  ]
}
```

- `temperature: 0`, `num_predict: 512`。

**24小时智能熔断截断（V5.0 核心新增）：**

1. 时间归一化清洗：正则匹配 `15小时前` / `3分钟前` / `1天前` → 换算绝对日期 `YYYY-MM-DD`。
2. 遍历 `orders` 数组，首次遇到 `time_str` 含 **"天前"** 关键词 → 触发 `STOP_SCROLL_AND_CLOSE` 信号。
3. 生产者接收信号 → 终止下滑 → 点击弹窗 `X` 关闭 → 流转下一单品。

**Ollama 冷启动补丁：**
- 启动时 `keepalive: -1` 预热，结束后 `keepalive: "0s"` 释放。

### 3.2 单元验收标准

```text
[ 输入测试数据集 ] → OpenCV 卡片拆分 → 信息熵/方差校验
  → 动态入口寻址(OCR特征匹配) → Qwen2.5-VL 多模态提取 → 时间归一化 → 24h熔断判定
```

| 样本 | 输入 | 预期行为 | 判定 |
|------|------|---------|------|
| 样本 1 | 纯白卡片（未加载） | 熵 < 1.0，方差 < 10，触发无效帧过滤 | 直接销毁，大模型调用 0 次 |
| 样本 2 | 含"超200人加购"文本的详情页截图 | OCR 命中特征 → 高斯偏移点击 → 唤醒弹窗 | 入口寻址成功 |
| 样本 3 | 实时销量弹窗截图（含 15 条订单，最后一条为"1天前"） | Qwen-VL 提取 15 条 → 第 15 条触发 24h 熔断 | 输出 14 条今日订单，第 15 条截断 |
| 样本 4 | 实时销量弹窗截图（含"回头客"标签订单） | Qwen-VL 提取 `is_repeat: true` | JSON 包含回头客标记 |
| 样本 5 | 含"3分钟前"、"15小时前"的混合时间戳 | 归一化后均为今日日期 | 全部归类为 `order_date = today` |

## 阶段四：时序分区存储与会话控制层（持久化与安全）

> 解决快照数据归属问题，新增微观实时订单流水表，防御海量时序数据写入性能危机。

### 4.1 开发执行步骤

**店铺会话初始化（Session Setup）：**
- 进店首页 → 截取顶部 → OCR 识别店铺名 → 绑定 `store_id`。
- 捕获商品总数基数（如"共 240 件商品"），初始化进度条。

**靶点池动态调度（V5.0 新增）：**
- 加载 `targets.yaml` 靶点池配置。
- Tier 1（核心监控）优先，Tier 2（宏观大盘）间歇穿插。
- `random.shuffle()` 随机打散执行顺序，杜绝固定周期特征。

**动态入口寻址 + 降级兜底（V5.0 新增）：**
- 详情页 OCR 扫描 `ENTRY_FEATURES` 特征词。
- 命中 → 高斯点击唤醒实时销量弹窗。
- 未命中 → 降级抓取"月销"文本 → Regex 提取 → 写入 `rolling_sales_history`。

**环境指纹热洗 + 网络隔离：**
- 动态随机变更 Android ID / IMEI / 序列号 / 蓝牙 MAC / Build.PROP。
- 飞行模式断网 3 秒重新附着基站获取全新公网 IP。

**硬件温控暂停 + 电池伪装：**
- 每 30 秒读取热 Zone 温度，> 42°C 暂停 5-10 分钟降温。
- 电池电量随机 40%-85%，伪装"未充电"状态。

**入店路径天然伪装：**
- 进店前首页随机点击 1-2 个热搜词假装浏览。
- 搜索爆款商品 → 详情页 → 点击进店（自然搜索推荐流量链）。

**时序分区数据库（DDL）落地：**
- `rolling_sales_history` 表按月物理分区 + 联合唯一索引 `(virtual_id, record_date)`。
- `competitor_realtime_orders` 表（V5.0 新增）+ 联合唯一索引 `(virtual_id, buyer_mask, sku_name, order_date)`。
- 写入采用 `INSERT IGNORE INTO` 协议。

### 4.2 单元验收标准

**测试算子 1（联合唯一索引约束 — 快照表）：**
- 并发 INSERT 两条相同 `(virtual_id, record_date)` 记录。
- 第二条必须被拒绝，抛出 Duplicate entry 异常。

**测试算子 2（联合唯一索引约束 — 订单流水表 V5.0 新增）：**
- INSERT 两条相同 `(virtual_id, buyer_mask, sku_name, order_date)` 记录。
- 第二条被 `INSERT IGNORE` 静默丢弃，不影响其他流水。

**测试算子 3（分区执行计划检查）：**
```sql
EXPLAIN SELECT * FROM rolling_sales_history WHERE record_date = '2026-05-28';
```
- 执行计划必须精确指向当月物理分区，严禁全表扫描。

**测试算子 4（靶点池随机打散 — V5.0 新增）：**
- 连续 5 次启动脚本，记录 Tier 1 店铺执行顺序。
- 验证每次顺序不同（`random.shuffle` 生效）。

**测试算子 5（24h 熔断 + 降级兜底 — V5.0 新增）：**
- 模拟入口特征未命中场景 → 系统降级抓取"月销"文本 → 正确写入 `rolling_sales_history`。
- 模拟命中"1天前"订单 → `STOP_SCROLL_AND_CLOSE` 信号抛出 → 生产者终止下滑。

## 阶段五：随动人机协同、全自动高可用与全链路联调（全面交付）

> 系统脱离凌晨定时窗口，在白天随动触发模式下，配合人机协同，在各种黑天鹅环境下保持极高数据可用性。

### 5.1 开发执行步骤

**随动触发调度（V5.0）：**
- 抛弃凌晨 2:00-4:00 定时窗口。
- 用户随时插线启动 → 环境预检 → 靶点池随机打散 → 随动采集。
- 单品页路径：进店 → 目标商品 → 动态入口寻址 → 点击唤醒弹窗 → Qwen-VL 提取 → 24h 熔断 → 关闭弹窗 → 下一商品。

**人机协同解围闸门：**
- 弹窗/验证码检测命中 → 终止手势注入 → 丢弃队列脏帧。
- 控制台高亮警告 + 主板蜂鸣 → 阻塞等待。
- 用户手机上真手操作 → 敲击 `Enter` → 脚本瞬间复活。
- 30 分钟无人处理 → 硬重置 × 3 → 失败则彻底停机。

**随动控制台进度条（V5.0 新增）：**
- 终端渲染动态进度条：店铺名称、百分比、实时速率（s/商品）、ETA、队列负载水位。

**24小时边界单店总控（V5.0 核心新增）：**
- 每次唤醒实时销量弹窗，Qwen-VL 执行语义提取。
- 时间归一化模块逐条清洗 `time_str`。
- 首次命中"天前"关键词 → 熔断阻断 → 关闭弹窗 → 单品采集完成。
- `INSERT IGNORE INTO competitor_realtime_orders` 批量写入今日订单。

**回话检查点断点续跑：**
- Ctrl+C → 消费者排空残留帧 → 序列化 `checkpoint.json`（含当前单品锚点）。
- 下次插线 → 交互提示续跑 → 自动跳过已采集商品。

**二次调度与补抓机制：**
- 每次任务结束后自动盘点数据完整率。
- 完整率 < 90% → 当日 8:30 创建补抓任务，标签 `Replenish_Snapshot`。

**全局改版数据强行熔断锁（UI Shift Kill-Switch）：**
- 连续 20 个商品卡片宽高比偏离 > 15% 或 Qwen-VL 连续 20 次输出异常 → 熔断。
- V5.0 新增触发条件：动态入口寻址连续 10 个商品未命中。

**时序空洞拟合（前端插值消费）：**
- 线性插值修复缺失日：`Value = (before + after) / 2`。
- ECharts 虚线 + 特殊色彩标记。

### 5.2 全系统终期验收红线（General Gatekeeping）

**地狱测试算子 1（弹窗 + 验证码综合拦截）：**
模拟系统弹窗/滑块验证码遮挡。

**地狱测试算子 2（改版熔断）：**
连续传入 20 张错位商品卡片。

**地狱测试算子 3（24h 熔断截断 — V5.0 核心新增）：**
构造含混合时间戳的实时销量弹窗数据（今日 × N + 1天前 × 1），验证第 1 条"天前"订单触发熔断，今日订单完整入库。

**地狱测试算子 4（动态入口寻址降级 — V5.0 新增）：**
- 构造不含 `ENTRY_FEATURES` 特征词的详情页截图。
- 验证系统正确降级为月销文本抓取。
- 构造含"超300人加购"的详情页截图 → 验证正确寻址 + 高斯点击。

**地狱测试算子 5（触控 MotionEvent 审计 — V5.0 新增）：**
- 500+ 次滑动 Pressure 非固定值，Size 非恒为 0。
- 1000 次点击坐标呈高斯圆斑分布。

**地狱测试算子 6（order_date 去重 — V5.0 新增）：**
- 同一 `(virtual_id, buyer_mask, sku_name, order_date)` INSERT 两次。
- `INSERT IGNORE` 第二条被静默丢弃。

**判定红线（全自动化与稳定性终极指标）：**

1. **绝对不允许**主进程崩溃抛出异常而停止工作。
2. 弹窗检测命中 → 生产者立即终止手势注入 → 本地蜂鸣声 → 控制台高亮警告。
3. 按下 `Enter` 后，脚本**瞬间无缝复活**，继续向下采集。
4. 改版数据强行熔断锁必须在第 **20 帧**异常发生后**毫秒级**触发。
5. 脚本立即强制终止手机滑动，**死锁数据库写入闸门**。
6. **24 小时熔断截断**在首条"天前"订单命中后立即触发，今日订单不丢单、昨日订单不混入。
7. Ctrl+C 优雅退出 → `checkpoint.json` 正确写入 → 下次插线检测到未完成任务 → 提供续跑选项。
8. 触控审计通过：Pressure 动态波动，Size 正态形变。
9. 热区审计通过：点击坐标呈高斯圆斑。
10. 端口隐匿通过：scrcpy 无特征泄露，KernelSU 通过 Anti-Taint 检测。
